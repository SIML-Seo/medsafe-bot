# 안전 정책

## fail-closed

약 특정 실패, token 누락·만료·불일치, 성분정보 부재·부분 파싱, DUR 규칙 조건 해석 실패, 카탈로그·품목 스냅샷 불완전, 페이지 무결성 실패는 녹색이 아니다. 핵심 검사를 완료하지 못하면 `failedTypes`와 구체적 `unresolved` 사유를 남기고 `UNCERTAIN`으로 반환한다.

`NO_KNOWN_FINDINGS`는 확인된 입력과 현재 핵심 검사 범위 안에서 등록된 finding을 찾지 못했다는 뜻일 뿐 안전 보장이 아니다. 서버는 임의 중단·복용 허가·용량 변경을 지시하지 않는다.

## 범위 보류

- `child`와 `elderly`: 연령별 DUR 미구현을 CAUTION finding으로 명시한다.
- 임신 `yes`: 임부 DUR 미구현을 CAUTION finding으로 명시한다.
- 연령 또는 임신 정보가 없으면 해당 범위가 보류됐음을 명시한다.
- 식품·건강기능식품·한약은 의약품 품목 완전일치가 없을 때 `OUT_OF_SCOPE`다.

## 응급 우선

호흡 불가·곤란·숨참, 의식소실, 경련, 아나필락시스, 심한 흉통, 입술 부종, 과량복용, 여러 알·다량 용기 복용, 자해 의도 표현은 119/응급실 안내로 우선 전환한다. 명시적 부정 표현은 그 증상 구절만 제거하며 이중부정과 뒤에 이어지는 양성 신호는 유지한다. 정의 질문과 현재 괜찮다고 명시한 과거 증상은 응급으로 오탐하지 않는다.

## 출력 방어

NFKC 정규화와 zero-width 제거 후 HTML·제어문자·프롬프트성 지시문을 정제한다. “안전합니다”, “복용해도 됩니다”, “끊으세요” 같은 단정 표현은 보수적 문구로 교체한다. 표준 디스클레이머와 범위 문구는 서버가 강제한다.

## 개인정보와 비밀

약 목록은 영속 저장하거나 요청 로그에 남기지 않는다. 공공데이터 키는 빌드 환경에서만 사용하고 runtime image에 포함하지 않는다. `.env`와 `.secrets/`는 Git·Docker build context에서 제외한다.

공개 PlayMCP read-only endpoint이므로 별도 사용자 인증은 두지 않는다. Origin 없는 서버 간 MCP 호출은 허용하고, Host allowlist와 논리 클라이언트·물리 ingress 요청률, JSON-RPC batch 상한, POST 처리 슬롯, HTTP 연결·헤더 시간 상한으로 남용을 제한한다. XFF는 설정된 신뢰 프록시 CIDR에 속한 socket peer에서만 사용하고, stateless endpoint의 GET·DELETE는 `405`로 닫는다.
