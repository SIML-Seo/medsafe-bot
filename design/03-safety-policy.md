# 안전 정책

## fail-closed 단일 정의

매핑 불확실, confirmation token 누락/불일치, DUR 응답 공백, 약 특정 실패, 페이지 수집 실패, 8종 중 일부 실패, 필드 매칭 실패, serviceKey 미설정, 실 API 타임아웃은 녹색이 아니다. 서버는 `UNCERTAIN` 또는 `CAUTION`으로 강등하고 약사/의사 확인 권장을 붙인다.

## 서버 강제 문구

모든 `check_medication_safety` 응답 하단에 다음 문구를 붙인다.

> 본 정보는 식약처 공개데이터 기반 일반 참고용이며 의사·약사의 진단·처방·복약지도를 대체하지 않습니다. 실제 복용·중단·변경은 반드시 의사 또는 약사와 상담하세요. 응급 증상 시 즉시 119.

스코프 문구:

> 이 결과는 건강기능식품·식품·한약·일부 의약품 정보를 포함하지 못할 수 있습니다.

## 금칙어 필터

서버 생성 텍스트에서 다음 단정 표현을 차단한다.

- 안전합니다
- 안심하세요
- 복용해도 됩니다
- 먹지 마세요
- 끊으세요
- 용량을 바꾸세요

필터에 걸리면 safe replacement로 교체하거나 테스트를 실패시킨다.

HTML 태그와 프롬프트성 문구는 user-visible text와 `structuredContent` 양쪽에서 escape/drop 처리한다.

## 응급 우선

사용자 입력 또는 displayName/context note에 다음 응급어가 포함되면 상호작용 조회보다 119/응급실 안내를 우선한다.

- 호흡곤란
- 의식저하
- 아나필락시스
- 심한 흉통
- 입술 부종

## 오매핑 방어

노란색/빨간색 finding에는 실제 매핑 제품명과 "이 약이 아니면 이 경고는 무시하세요"를 붙인다. 이미 처방받은 조합일 수 있으므로 임의 중단 금지 문구를 함께 붙인다.

## 개인정보

- 약 목록은 영속 저장하지 않는다.
- 요청 payload는 파일 로그에 남기지 않는다.
- OAuth 사용자 식별자는 현재 tool 입력/판정에 쓰지 않는다.
- serviceKey는 환경변수로만 읽는다.
- DUR 공공 API는 HTTPS endpoint만 사용한다.
